Installation

Crew builds on macOS, Linux, and Windows.

One-liner (macOS & Linux)

curl -sSfL https://raw.githubusercontent.com/ashishtyagi10/crew/main/install.sh | sh

Installs the prebuilt crew binary to ~/.local/bin. Set INSTALL_DIR to choose another location.

With cargo (any platform with Rust)

cargo install --git https://github.com/ashishtyagi10/crew crew-app

GitHub Releases

Download the latest archive for your platform from the Releases page, extract it, and move the crew binary to a directory on your PATH:

Build from Source

git clone https://github.com/ashishtyagi10/crew.git cd crew cargo build --release -p crew-app # Binary is at target/release/crew

Updating

Type /update in the input bar to fetch and install the latest release.

Build & Run

cargo run --release -p crew-app

Shells launch as your $SHELL login shell, so your full config and plugins load. The window opens with panes auto-tiled into a near-square grid and a docked command bar at the bottom.

Panes

Panes auto-tile into a near-square grid. Each pane has a title bar (top row) showing its index, the program-set title (often the cwd), and right-aligned status glyphs. The focused pane has a near-white border and a bright block cursor; unfocused panes are grey with a dim cursor.

GlyphMeaning
⇡Nviewing scrollback, N lines back from the live bottom
●new output in an unfocused pane
!the program rang the bell
»receiving broadcast (synchronized) input

Click a pane to focus it (click the input bar to focus that); double-click a pane to toggle zoom. Inside a terminal pane, all other keys (arrows, Home/End, PageUp/Down, Ctrl+C, Shift+Tab, …) pass through to the program.

The Input Bar

The docked command bar at the bottom focuses with Cmd+I. Anything that isn't a slash command or cd is sent to the focused terminal. The bar's legend shows Crew's current working directory (~-abbreviated).

Autosuggest & History

  • Autosuggest — fish-style ghost text from history; Tab/→ accepts it.
  • History — Up/Down recall previous lines, persisted to $XDG_CONFIG/crew/history across sessions.

cd Completion

Typing cd <partial> ghost-completes the first matching subdirectory; Tab/→ accepts it. $VAR/${VAR} are expanded (e.g. cd $HOME/src). cd - toggles back to the previous directory, and the working directory is restored on the next launch. New shells (Cmd+T / /shell) open in the bar's current directory.

Editing & Status

  • Ctrl+W deletes the last word; Ctrl+U clears the line.
  • Status flashes — transient messages (e.g. "copied 12 lines", "cd: no such directory") appear briefly on the input card's bottom border.

Multi-Agent (/crew)

/crew opens a pane where independent headless CLI coding agents — claude, codex, and opencode — message each other to work a task you give them, in any direction.

On open, the broker probes which agent CLIs are installed and registers only those (missing ones are skipped, and it says so). Type a task and press Enter; prefix @<agent> to choose who starts.

The relay protocol

Each agent receives a clean, normalized message — never another agent's raw CLI output — and can route in three ways:

  • begin a reply with TO <agent>: <message> to hand off to a peer;
  • reply DONE (or DONE: <answer>) to end the thread — the explicit no-reply signal;
  • anything else is a plain reply that bounces back to the sender.

This proves out as A→B (claude hands to codex), B→A (codex replies back), and a 3-way relay (claude → codex → opencode, with the answer relayed back to claude). A live run:

claude → codex TO codex: reply with exactly the word PONG codex → claude PONG claude → codex [done]

Guardrails

  • Loop guard — every message carries a hop counter; past the limit (default 6) the broker drops the thread and logs that it stopped, so a relay can never loop forever.
  • Per-call timeout — a hung agent is killed and logged (default 180s); the broker moves on instead of blocking.
  • Isolation — agents run in a broker subprocess over Crew's JSON plugin protocol, so slow calls stay off the render thread; an adapter normalizes each agent's stdout before it is shown or relayed.
  • Observability — every hop is logged in the pane as from → to with the reply.

Adding an agent

Discovery and routing are agent-agnostic. To add a fourth agent, write one adapter constructor in crates/crew-plugin/src/broker/agents.rs and register it in known_adapters — the routing engine is untouched.

Clipboard

  • Cmd+C copies the focused terminal's visible screen to the system clipboard.
  • Cmd+V pastes into the focused surface (terminal, input bar, or chat). For terminals it uses bracketed paste when the program enabled it.
  • Programs can copy to the system clipboard via OSC 52.

Scrollback

Mouse wheel or Shift+PageUp/Shift+PageDown scroll a pane's history; an amber ⇡ in the title bar marks that you're viewing scrollback. /find <text> scrolls back to the most recent line containing the text (smart case: case-insensitive unless the term has an uppercase letter); a miss reports on the status line. Typing returns to the bottom.

Click a PANES row to focus that pane (double-click to zoom it).

Settings

/settings opens a form (two columns on a wide pane, one when narrow):

  • Font family — type-to-search over installed monospace families.
  • Font size, Nav width, Show nav.

Settings persist to $XDG_CONFIG/crew/config.toml and apply live on Save.

Theme

The canvas is pure black; terminal content shows its natural ANSI colors. The accent green is reserved for chrome (borders, the CREW wordmark, the command palette). A configurable theme is future work.

Architecture

Crew is a from-scratch GPU terminal. Every cell is drawn on the GPU; panes have SDF rounded borders. The terminal model is built on alacritty_terminal + portable-pty. In-pane UI (the settings form, command palette, and help overlay) is laid out with ratatui widgets into a Buffer and converted to GPU cells.

Crate Structure

Crew is a Cargo workspace with four crates:

CratePurpose
crew-appWindow, panes, input, in-pane UI
crew-renderGPU rendering (wgpu + glyphon)
crew-termPTY + terminal grid (alacritty_terminal + portable-pty)
crew-pluginChat / agent plugins

Hard Rules

  • Every .rs file stays ≤200 lines — no exceptions.
  • cargo clippy --workspace --all-targets is warning-free.

Contributing

The short version:

  1. Fork and clone the repo
  2. Create a feature branch
  3. Make your changes, keeping every .rs file ≤200 lines and clippy clean
  4. Push and open a PR against main
