# Crew Home: Welcome Art + Bottom Input Box Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Give Crew a home screen ‚Äî a fancy `CREW` ASCII banner in the main area
(the empty state) and a docked bottom **input box** (command bar) the user can focus
and type into.

**Architecture:** Both render as `CellView`s (no `crew-render` change). The welcome
art is drawn into the content area when there are no grid panes. The input box is a
docked bottom surface (like the sidebar is docked-left): `chrome.rs` reserves an
`input_h` strip at the bottom; `render.rs` shrinks the sidebar/content height by it
and pushes an input-box `PaneScene`. Input focus is a new `CrewApp.input.focused`
flag toggled by `Cmd+I`; while focused, keys edit the box and `Enter` sends the line
to the focused terminal. Startup lands on the welcome (no auto-terminal); closing the
last pane returns to the welcome rather than exiting (`Cmd+Q`/`Ctrl+Q` still quit).

**Tech Stack:** Rust 2021, existing crates only.

## Global Constraints
- **`crew-render` untouched** (everything is `CellView`).
- **Every `.rs` ‚â§ 200 lines (HARD).**
- **Reserved keys = Super-chords only** (adding `Cmd+I`); non-Super keys go to the
  focused pane OR, when the input is focused, to the input box.
- `cargo clippy --workspace --all-targets` ZERO warnings; **no `#[allow]`**.
- Panels follow the titled-card pattern (`boxdraw::titled_box`) ‚Äî the input box is a
  titled card `‚ï≠‚îÄ INPUT ‚îÄ‚ïÆ`.
- Gate: compile + clippy clean + `cargo test` + `timeout 6 cargo run -p crew-app` ‚Üí exit 124.

### Task 2: bottom input-box layout + rendering (TDD)

**Files:** Create `crates/crew-app/src/inputbar.rs`; modify `main.rs`, `chrome.rs`,
`render.rs`, `app.rs` (CrewApp field).

**Interfaces:**
```rust
// chrome.rs ‚Äî reserve a bottom strip; sidebar & content shrink by input_h.
pub const INPUT_PAD: f32 = 6.0;
pub fn input_h(cell_h: f32) -> f32 { cell_h * 3.0 + INPUT_PAD } // ~3 rows for the card
pub fn inputbar_rect(sw: f32, sh: f32, ih: f32, gap: f32) -> Rect; // full width, bottom, gap-inset
// content_rect/sidebar_rect gain an `ih` param subtracted from their height.

// inputbar.rs
pub struct InputBar { pub text: String, pub focused: bool }
impl InputBar {
    pub fn new() -> Self;                                  // empty, not focused
    pub fn cells(&self, cols: u16, rows: u16) -> Vec<CellView>; // titled card "INPUT" + "> text"
    pub fn submit(&mut self) -> Option<String>;            // take text if non-empty, clear
}
```
- `chrome.rs`: `sidebar_rect`/`content_rect` shrink their height by `ih`
  (`h = sh - ... - ih`). Add `inputbar_rect(sw, sh, ih, gap)` = full-width strip at the
  bottom (`x: gap, y: sh - ih + ... `), gap-inset like the sidebar. Update chrome tests.
- `inputbar.rs::cells`: draw a `boxdraw::titled_box` titled `INPUT` (accent if
  `focused`, else border color), with `"> " + text` on the inner row. Truncate to width.
- `app.rs`: add `pub(crate) input: InputBar` to `CrewApp` (derive Default ‚Äî give
  `InputBar` a `#[derive(Default)]`-compatible `Default` or `impl Default`).
- `render.rs` `build_frame`: compute `ih = chrome::input_h(ch)`; pass it through to
  `content_rect`/`sidebar_rect`/`grid_rects` (height now excludes the input strip);
  push an input-box `PaneScene` at `chrome::inputbar_rect(...)` with
  `cells = self.input.cells(in_cols, in_rows)`. `pane_at_cursor` must also use the
  shrunk content rect.

### Task 3: input focus + editing + submit (build-run-observe)

**Files:** Modify `app.rs` (`handle_super_chord`), `handler.rs` (key routing), `inputbar.rs` (`on_key`).

**Interfaces:**
- `inputbar.rs`: `pub fn on_key(&mut self, key: &winit::event::KeyEvent) -> Option<String>`
  ‚Äî edit the buffer (printable char append, Backspace pop, Space) reusing the same
  reducer style as `chatlayout::input_reduce`; on `Enter` return `submit()`.
- `app.rs` `handle_super_chord`: `"i" => { self.input.focused = !self.input.focused; }`.
- `handler.rs` `KeyboardInput`, non-Super branch: if `self.input.focused`, route the
  key to `self.input.on_key(&event)`; if it returns `Some(line)`, send it to the
  focused terminal pane (`write_all(line + "\n")` to its `TermPane.input`, mirroring
  `send_to_label`), and DON'T also send to a pane. `Esc` while focused ‚Üí
  `self.input.focused = false` (handle before/within the input branch). When the input
  is NOT focused, keep today's pane routing unchanged.
